# **CONTRIBUTING.md**  
### *Guidelines for Contributing to the Plasma‑Resonance Metric Engine (PRME) Project*

Thank you for your interest in contributing to the Plasma‑Resonance Metric Engine (PRME).  
This project is an open, speculative exploration of resonance‑based spacetime manipulation using structured plasma configurations.  
It is not a proven technology, and contributions should reflect the project’s scientific, exploratory, and collaborative spirit.

The goal of this repository is to provide a structured, transparent foundation for interdisciplinary research, critique, and refinement.

---

# **1. Project Philosophy**

The PRME project is guided by the following principles:

- **Openness:** All contributions remain open under the AGPLv3 license.  
- **Scientific Integrity:** Claims must be grounded in known physics or clearly identified as speculative.  
- **Constructive Critique:** The project welcomes rigorous analysis, including challenges to assumptions.  
- **Interdisciplinary Collaboration:** Plasma physics, general relativity, nonlinear dynamics, and advanced computation are all relevant domains.  
- **Humility:** The PRME is a conceptual framework, not an established physical model.

# **3. Contribution Process**

### **3.1 Fork the Repository**
Create your own fork of the project to work on changes.

### **3.2 Create a Feature Branch**
Use a descriptive branch name, such as:

```
feature/plasma-stability-analysis
fix/typo-in-metric-section
proposal/resonance-modeling
```

### **3.3 Make Your Changes**
Ensure your contributions:

- are clearly written  
- cite relevant literature where appropriate  
- distinguish between established physics and speculation  
- maintain the tone and structure of the project  

### **3.4 Submit a Pull Request**
Include:

- a clear description of your contribution  
- the motivation behind it  
- any relevant references  
- any assumptions or limitations  

Pull requests are reviewed for clarity, scientific grounding, and consistency with the project’s goals.

---

# **4. Standards for Scientific Rigor**

Contributions should:

- avoid unsupported claims  
- avoid exotic matter assumptions unless explicitly justified  
- avoid violations of relativity or conservation laws  
- clearly separate speculation from established physics  
- reference credible scientific sources when possible  

This project values **quality over quantity**.

---

# **5. Code of Conduct**

All contributors are expected to:

- engage respectfully  
- critique ideas, not people  
- avoid dismissive or hostile behavior  
- support an environment of curiosity and collaboration  

This project is a space for exploration, not gatekeeping.

---

# **6. Licensing**

All contributions are governed by the **AGPLv3 license**.  
By submitting a pull request, you agree that your contribution will be released under this license and remain permanently open.

---

# **7. Questions or Proposals**

If you have:

- a new idea  
- a critique  
- a question  
- a proposed extension  

…you may open an **Issue** in the repository.  
Please provide as much clarity and context as possible.

---

# **8. Final Note**

The PRME is an early‑stage theoretical framework.  
It is not expected that contributors “prove” the concept.  
The purpose of this project is to explore, refine, challenge, and expand the idea collaboratively and transparently.

Thank you for helping advance this open scientific effort.
